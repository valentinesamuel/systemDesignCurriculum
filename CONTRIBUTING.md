# Contributing to The Staff Path

Thank you for considering a contribution. This is a community-maintained curriculum and every improvement helps engineers level up.

---

## What's welcome

- **Typo and grammar fixes** — open a PR directly.
- **Factual corrections** — if a technical claim is wrong, open an issue first with a brief explanation, then submit a fix PR.
- **Clarity improvements** — if a Problem Statement is ambiguous in an unintentional way, open an issue to discuss before rewriting.
- **New chapters** — if you want to add a company arc in an industry not yet covered, open an issue first with a brief pitch: industry, company name, 4–6 chapter titles, and the core system concepts you'd cover.
- **Translations** — open an issue to claim a language before starting so work isn't duplicated.
- **Broken links** — file a PR with the fix.

## What's not welcome

- **Reference solutions** — their absence is intentional. PRs that add answer keys will be closed.
- **Opinionated rewrites** — if you'd design a chapter differently, that's valid. The curriculum is intentionally ambiguous. Redesigning chapters to match your preferred approach is not a contribution.
- **Reducing difficulty** — chapters are calibrated to their level arc. Simplifications that lower the ceiling will not be merged.

---

## How to submit a PR

1. Fork the repo and create a branch: `git checkout -b fix/your-description`
2. Make your changes.
3. Verify: does the chapter still follow the [chapter format](CLAUDE.md#chapter-format-strict--every-chapter-must-follow-this-exactly)?
4. Open a PR with a clear title and a one-sentence description of what changed and why.

## Opening an issue

Use issues for:
- Factual errors (include the source you'd cite for the correction)
- New chapter proposals (see "New chapters" above)
- Questions about how to use the curriculum

---

## Code of conduct

Be direct and specific in feedback. Assume good intent. Critique the work, not the person.
